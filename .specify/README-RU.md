# Структура документации Spec Kit

Этот каталог содержит всю документацию проекта в соответствии с методологией Spec Kit.

## Структура каталогов

```
.specify/
├── memory/              # Общепроектная документация и принципы
│   └── constitution.md  # Основные принципы и стандарты проекта
├── scripts/             # Скрипты автоматизации для spec-driven workflow
│   └── bash/            # Bash скрипты
│       ├── check-prerequisites.sh    # Проверка предварительных условий для фичи
│       ├── common.sh                 # Общие функции
│       ├── create-new-feature.sh     # Создание структуры новой фичи
│       ├── setup-plan.sh             # Инициализация плана реализации
│       └── update-agent-context.sh   # Обновление контекстных файлов агентов
├── specs/               # Спецификации фич
│   └── NNN-feature-name/            # Каталоги отдельных фич
│       ├── spec.md                   # Спецификация фичи
│       ├── tasks.md                  # Разбивка на задачи
│       ├── plan.md                   # План реализации
│       ├── data-model.md             # (Опционально) Модель данных
│       ├── research.md               # (Опционально) Заметки исследования
│       ├── quickstart.md             # (Опционально) Руководство быстрого старта
│       ├── checklists/               # (Опционально) Чек-листы
│       └── contracts/                # (Опционально) API контракты
└── templates/           # Шаблоны документов
    ├── spec-template.md
    ├── tasks-template.md
    ├── plan-template.md
    ├── checklist-template.md
    └── agent-file-template.md
```

## Использование

### Создание новой фичи

Используйте скрипт create-new-feature для инициализации новой фичи:

```bash
./.specify/scripts/bash/create-new-feature.sh "Описание фичи" --short-name "feature-name"
```

### Проверка предварительных условий

Перед работой над фичей проверьте предварительные условия:

```bash
./.specify/scripts/bash/check-prerequisites.sh --json
```

### Работа с агентами Spec Kit

Доступны следующие агенты через GitHub Copilot:

- `/speckit.specify` - Создать спецификацию новой фичи
- `/speckit.clarify` - Прояснить неоднозначности в спецификации
- `/speckit.plan` - Сгенерировать план реализации
- `/speckit.tasks` - Разбить на выполнимые задачи
- `/speckit.implement` - Реализовать фичи
- `/speckit.analyze` - Анализировать консистентность
- `/speckit.taskstoissues` - Конвертировать задачи в GitHub issues

## Стандарты именования файлов

Все спецификации фич следуют стандартному именованию:

- **spec.md** - Спецификация фичи
- **tasks.md** - Разбивка на задачи
- **plan.md** - План реализации
- **data-model.md** - Модель данных (опционально)
- **research.md** - Заметки исследования (опционально)
- **quickstart.md** - Руководство быстрого старта (опционально)

## Конституция

Конституция проекта (`memory/constitution.md`) определяет основные принципы и стандарты, которым должны следовать все фичи. Это авторитетный источник управления проектом, который не может быть переопределен спецификациями отдельных фич.

## Шаблоны

Шаблоны в каталоге `templates/` предоставляют стандартизированные структуры для всей документации. Используйте эти шаблоны при ручном создании новой документации или позвольте агентам Spec Kit использовать их автоматически.
